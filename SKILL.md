## 工作流程

### 模式一：测试阶段（不实际投递）

```
步骤1：点击职位卡片 → 截图
步骤2：API 识别岗位信息 → 判断是否符合条件
步骤3：符合条件 → 生成招呼语（保存，不发送）
步骤4：不符合条件 → 记录理由
步骤5：滚动到下一个岗位
```

**输出：** 岗位匹配表格 + 招呼语预览

---

### 模式二：正式投递

```
步骤1：点击职位卡片 → 截图
步骤2：API 识别岗位信息 → 判断是否符合条件
       - 不符合 → 记录理由，跳至步骤5
       - 符合 → 继续步骤3
步骤3：生成招呼语 → 复制到剪贴板 → 点击"立即沟通" → 点击"继续沟通"
步骤4：点击聊天输入框 → 粘贴招呼语 → 等待 0.5 秒 → 按 Enter 发送 → 点击返回按钮
步骤5：移动到职位卡片位置 → 滚动 180px → 等待 1 秒 → 回到步骤1
```

**输出：** 实际投递 + 投递记录表格

- 每日最多投递 20 个岗位（避免被平台限制）
- 同一公司不重复投递
- 投递前截图保存岗位信息

## 故障处理

| 问题 | 解决方案 |
|------|----------|
| 坐标不准确 | 更新 `config/coordinates.json` |
| 滚动距离不对 | 调整 `config/coordinates.json` 中的 scrollDistance |
| API 限流 | 增加等待时间，减少并发 |
| 投递失败 | 检查网络连接，重试或跳过 |

# 使用 OpenClaw Agent 方式执行

由于 BOSS 直聘自动投递需要调用 OpenClaw 的 browser 和 image 工具，最佳使用方式是通过 OpenClaw 对话执行，而不是独立 Python 脚本。

---

## 方式一：通过 OpenClaw 对话使用（推荐）

### 测试模式

```
用户：测试 BOSS 直聘自动投递，测试 5 个岗位

Agent: 
1. 打开 BOSS 直聘岗位列表页
2. 依次识别 5 个岗位（使用 browser snapshot + qwen3.5-plus 图像识别）
3. 判断是否符合条件
4. 生成招呼语（不发送）
5. 输出匹配表格
```

### 正式投递

```
用户：开始正式投递，投递 6 个岗位

Agent:
1. 打开 BOSS 直聘岗位列表页
2. 依次识别并投递 6 个岗位
3. 每个岗位：截图→识别→判断→生成招呼语→发送→记录
4. 输出投递结果表格
```

---

## 方式二：Python 脚本（需要 API 集成）

如果需要使用 Python 脚本独立运行，需要：

1. **配置 API Key**
   ```bash
   export DASHSCOPE_API_KEY="sk-xxx"
   ```

2. **安装依赖**
   ```bash
   pip install dashscope pyautogui pyperclip pillow
   ```

3. **运行脚本**
   ```bash
   python scripts/boss_auto_apply.py --mode test --count 5
   ```

---

## 当前推荐方式

**使用 OpenClaw 对话方式执行**，因为：
- ✅ 可以直接使用 browser 工具
- ✅ 可以直接使用 image 工具（qwen3.5-plus）
- ✅ 无需额外配置 API key
- ✅ 错误处理更完善

---

## 执行流程（OpenClaw 方式）

### 步骤 1：打开 BOSS 直聘

用户手动打开 BOSS 直聘岗位列表页，或使用 browser 工具打开。

### 步骤 2：开始执行

```
用户：开始测试 BOSS 直聘自动投递，测试 3 个岗位
```

### 步骤 3：Agent 执行

```
Agent 执行流程：
1. browser snapshot → 获取页面信息
2. 点击第一个职位卡片
3. browser screenshot → 截图
4. image (qwen3.5-plus) → 识别岗位信息
5. 判断是否符合条件
6. 生成招呼语
7. 测试模式：显示招呼语
8. 正式模式：点击立即沟通→发送招呼语
9. 滚动到下一个岗位
10. 重复步骤 2-9
```

### 步骤 4：输出结果

```
Agent 输出：
- 投递结果表格
- 招呼语预览（测试模式）
- CSV 文件保存位置
```
